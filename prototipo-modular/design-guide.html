Princípios

Extraídos do visual já implementado no protótipo modular.

Clareza acima de decoraçãoInformação primeiro; ornamento só quando ajuda.
Hierarquia por tipografia e espaçamentoPeso, tamanho e whitespace — não mais bordas.
Interface compactaCabeçalhos baixos, inputs ~32px, respiro entre grupos.
Aproveitamento inteligente da telaDuas colunas, grades densas, sem desperdício vertical.
Poucas bordasBordas suaves e baixo contraste nos cards internos.
Sombras discretasApenas sombras leves azuladas (navy/accent).
Componentes reutilizáveisReusar classes e padrões; não inventar variantes.

Fazer / Não fazer

Contrato do protótipo — útil para pessoas e para o Agent.

Fazer

  • Reutilizar componentes e classes existentes
  • Manter tokens de 01-base.css
  • Manter identidade visual Processo Ágil
  • Aproveitar o espaço da tela
  • Priorizar clareza e hierarquia tipográfica
  • Editar em partials/, css/, js/ + build
  • Seguir os padrões já implementados

Não fazer

  • Não usar Tailwind
  • Não usar Bootstrap
  • Não usar React
  • Não criar padrões visuais novos sem necessidade
  • Não alterar IDs
  • Não modificar JavaScript sem solicitação
  • Não usar excesso de bordas
  • Não criar cabeçalhos altos
  • Não exagerar em cores ou sombras
  • Não duplicar componentes
